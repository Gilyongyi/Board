개인프로젝트

프로젝트 내용 : 게시판 만들기

기간 : 2016.10.24(월) ~ 2016.11.10(목)

요구사항

  - 기본적인 CRUD, 페이징 처리, 검색, 파일업로드, 댓글

특징

  • Model

    1. MVC에서 Model은 Bean으로 이루어져 있으며 데이터를 관리함.

    2. 요청에 맞는 컨트롤러의 메소드를 실행하여 Domain 과 DBMS 사이의 데이터를 주고 받음

    3. jdbc.properties File을 통해 JDBC접속정보를 저장하여 접속함.

    4. MyBatis를 이용하여 DML을 관리함.

    5. Oracle 과 MySQL 두가지의 DBMS를 사용하여 두 종류의 쿼리 작성

    6. DB Modeling은 exERD를 사용하여 논리모델링과 물리모델링을 거쳐 SQL문 생성

  • View

    1. MVC에서 View는 HTML을 parsing하여 Controller에서 전달된 데이터를 삽입하여 보여줌.

    2. Single Page Application (SPA) / 미리 parsing 된 HTML에 데이터만 삽입하여 리로딩이 없는 페이지를 구현함. ( 동적인 화면 구성 )

    3. HTML 만을 이용하여 Front-end 와 Back-end를 완벽하게 구분하여 View만 바꾸어준다면 하이브리드 웹앱으로 사용 가능함.

    4. 무한스크롤 페이징 처리

  • Controller

    1. MVC에서 Controller는 Bussiness Losic을 수행하며 대부분 Spring Framework에서 interface로 구현되어 있음.

    2. Front-end 에서 Request 를 하게되면 DispatcherServlet에서 받아 HandlerMapping을 통해 알맞은 컨트롤러의 메소드를 호출함

    3. Controller의 Method에서 GSON library를 사용하여 데이터를 JSON으로 바인딩 후 View에 넘겨줌.

  • ETC...

    1. Build Framework ( Gradle ) 을 사용하여 Dependency library 관리 (Build.gradle)

    2. NPM (Node Packaged Modules) 을 사용하여 필요한 패키지 설치하여 사용 (Package.json)

    3. 회원가입을 필요로 하지 않는 자유 게시판 ( 모든 권한은 글쓴이에게 있음 )

사용 기술 / Tool / DBMS

기술

  - Java , HTML5, CSS3, JavaScript, jQuery, ajax, MyBatis, Handlebars, Bootstrap

Tool

  - Eclipse, Gradle, exERD, MySQL Workbench, Oracle Developer, NPM

DBMS

  - Oracle11g, MySQL
